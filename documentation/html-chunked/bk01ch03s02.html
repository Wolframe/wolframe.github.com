3.2. Listener Configuration

The listener configuration is defined in the Listener section. The listener specifies the set of accepted connections and their attributes. It also defines some global settings listed in the following table:

Table 3.3. Global listener settings

NameDescription
MaxConnectionsThe maximum number of total simultaneous connections (clients). When set to 0 then the default is used. The Default is the operating system limit.
ThreadsNumber of threads for serving client connections


Here is an example configuration of the global settings: 

Listener
{
    MaxConnections 12
    Threads 7
}

The listener allows 2 different types of connections to configure. The types are also the section names where they are configured in the listener configuration. One is Socket that describes a plain connection. The other is SslSsocket that describes a connection with transport layer encryption based on SSL/TLS. In the following 2 sections they are introduced:

3.2.1. Socket

In subsections Socket of the section Listener of the configuration we describe plain connections based on TCP. The follwing table describes the attributes you can set for plain TCP connections:

Table 3.4. Socket settings

NameDescription
AddressListening address (IPv4 or IPv6) of the server. '127.0.0.1' and '::1' stand for the loopback address (IPv4 an IPv6 respectively). The listener wildcards '*' or '0.0.0.0'/'::' (IPv4 or IPv6) are also accepted.
PortConnection port
IdentifierIdentifier that can be reference in authorization functions to classify connections and do authorization based on it.
MaxConnectionsThe maximum number of simultaneus connections for this port.


The following configuration shows an example plain TCP connection definition: 

Listener
{
    Socket
    {
        address     localhost
        port        7662
        identifier  "Interface 1"
    }
}

3.2.2. SslSocket

In subsections SslSocket of the section Listener of the configuration we describe connections connection with transport layer encryption based on SSL/TLS. The follwing table describes the attributes you can set for SSL/TLS connections. The first 4 attributes are the same as for TCP/IP plain:

Table 3.5. SslSocket settings

NameDescription
AddressListening address (IPv4 or IPv6) of the server. '127.0.0.1' and '::1' stand for the loopback address (IPv4 an IPv6 respectively). The listener wildcards '*' or '0.0.0.0'/'::' (IPv4 or IPv6) are also accepted.
PortConnection port
IdentifierIdentifier that can be reference in authorization functions to classify connections and do authorization based on it.
MaxConnectionsThe maximum number of simultaneus connections for this port.
CertificateFile with the SSL certificate
KeyFile with the SSL key
CAdirectoryDirectory holding the CA certificate files
CAchainFileSSL CA chain file
verifyON/OFF switch to enabe/disable client certificate verification


The following configuration shows an example SSL connection definition: 

Listener
{
    SslSocket
    {
        address        localhost
        port           7662
        identifier     "Interface 1"
        maxConnections 2
        certificate    SSL/wolframed.crt
        key            SSL/wolframed.key
        CAchainFile    SSL/CAchain.pem
        verify         OFF
    }
}
